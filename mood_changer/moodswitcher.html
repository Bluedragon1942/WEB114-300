Quesitons:

    Why is textContent safer than innerHTML?

  1. The reason why textContent is safer than innerHTML is because textContent treats everything as plain text and doesn't parse or execute the HTML or scripts. This basically means that if a malious attacker were to have control and make the user input contain malicious code, it will be displayed harmlessly instead of running. But if you were to use innerHTML, it will interpret the content as HTML, which can allow attackers to inject and execute scripts if the input isn’t properly sanitized. (As said in the video, suseptable to cross-attacks using JavaScript.)

    2. How are HTML elements connected to JavaScript variables using DOM selection?

  2. HTML elements are connected to the JavaScript variables through the DOM selection methods (like document.getElementById, querySelector, or getElementsByClassName) which locate elements within the page and return references to them. These references can/are stored in variables which allows JavaScript to read or modify the element’s content, attributes, or styles directly.

    3. What would happen to the text box with id="text" if I added this line to the code above it:
    text.textContent = "Hello"

  3. It will modify that element with the id associated to the element to replace whatever text to Hello. Any text that was previously inside the element would be overwritten, and since textContent is used, the text will be inserted as plain text without interpreting any HTML.
